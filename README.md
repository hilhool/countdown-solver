# Countdown Puzzle Solver

Обучение модели `google/gemma-3-1b-it` решать задачу Countdown с использованием знаний модели-учителя из другого семейства (Qwen3-8B).

**Пример задачи:**
```
Числа: 75, 80, 90, 24
Цель:  61
Ответ: 90 - 80 + 75 - 24
```

## Подход

### 1. Синтетические данные (`1_generate.py`)
- 300k примеров, сгенерированных брутфорсом с проверкой через `Fraction`-арифметику
- Сложность: 2 оператора (40%), 3 (40%), 4 (20%)
- Числа от 1 до 99, таргеты от 1 до 999
- Глобальная дедупликация по паре (числа, таргет)

### 2. Дообучение модели (`2_train.py`)
- **Базовая модель:** `google/gemma-3-1b-it`
- **Учитель:** `Qwen/Qwen3-8B` (другое семейство, разные токенизаторы)
- **Метод:** SFT + LoRA (r=128, alpha=128)
- Loss только на уравнениях (completion-only), промпт маскируется
- 150k примеров, 2 эпохи, LR=2e-4 с cosine scheduler

### 3. Инференс (`3_inference.py`)
- **Первый проход:** beam search (5 лучей), температура 0.1
- **При неудаче:** 15 сэмплов с температурой 0.7, majority voting
- **Fallback:** исчерпывающий brute-force с таймаутом 5 сек

## Воспроизведение

**Требования:**
- GPU с 10+ GB VRAM (NVIDIA, CUDA 12+)
- Python 3.10+

**Установка:**
```bash
pip install -r requirements.txt
```

**Запуск:**
```bash
# 1. Генерация данных (~20 мин)
python 1_generate.py

# 2. Обучение (~10 часов на RTX 4060 Ti 16GB)
python 2_train.py

# 3. Инференс (~2 часа)
python 3_inference.py
```
